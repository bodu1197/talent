# Claude Code MCP 자동 사용 규칙

## 🚨 CRITICAL: 항상 MCP 우선 사용

### 필수 MCP 자동 사용 규칙

#### 1. Context7 (최우선 - ALWAYS)
**트리거:**
- 라이브러리/프레임워크 언급 (Next.js, React, TypeScript 등)
- "문서", "공식", "최신", "how to", "방법" 등의 키워드
- 코드 패턴 질문
- API 사용법 문의

**행동:**
- 무조건 먼저 context7로 최신 공식 문서 확인
- 답변 전에 반드시 실행
- 예: "use context7 to check latest Next.js 15 patterns"

**절대 금지:**
- ❌ Context7 없이 라이브러리 코드 작성
- ❌ 기억에만 의존한 API 사용법 제시
- ❌ "아마도", "보통" 같은 불확실한 답변

---

#### 2. Next.js DevTools (자동 체크)
**트리거:**
- 작업 시작 시 (첫 메시지)
- "에러", "안 돼", "작동 안 함", "문제" 키워드
- 코드 수정 완료 후
- 배포 전

**행동:**
- 자동으로 get_errors 호출
- 현재 런타임 에러 상태 확인
- 하이드레이션 에러, 빌드 에러 등 모두 체크

**절대 금지:**
- ❌ 에러를 추측으로 판단
- ❌ 실제 에러 확인 없이 해결법 제시

---

#### 3. Brave Search (불확실하면 즉시)
**트리거:**
- 에러 메시지 발견
- 2025년 1월 이후 정보 필요
- 특정 버전 호환성 문제
- "최신", "요즘", "지금" 키워드
- 생소한 에러나 이슈

**행동:**
- Stack Overflow, GitHub Issues 우선 검색
- 최신 해결법 확인
- 여러 출처 크로스 체크

**절대 금지:**
- ❌ 불확실한 정보로 답변
- ❌ 오래된 해결법만 제시

---

#### 4. Node.js Debugger (런타임 확인)
**트리거:**
- "디버깅", "왜", "값이", "undefined" 등
- 변수 값 확인 필요
- 런타임 동작 이해 필요
- 로직 추적 필요

**행동:**
- 실제 런타임에서 변수 값 확인
- 브레이크포인트 설정하여 단계별 확인
- 실제 실행 흐름 추적

---

#### 5. Playwright (모든 테스트)
**트리거:**
- "테스트" 키워드
- 기능 구현 완료 후
- 중요 기능 작업 시

**행동:**
- 자동으로 E2E 테스트 생성
- 사용자 시나리오 기반 테스트
- 크로스 브라우저 고려

---

#### 6. Sequential Thinking (복잡한 작업)
**트리거:**
- 아키텍처 설계
- 5단계 이상 작업
- "설계", "구조", "어떻게 구현" 등
- 복잡한 리팩토링

**행동:**
- 체계적으로 단계별 사고
- 각 단계별 검토
- 트레이드오프 분석

---

#### 7. GitHub (버전 관리)
**트리거:**
- "PR", "커밋", "푸시" 키워드
- 배포 준비
- 코드 리뷰 요청

**행동:**
- 커밋 메시지 자동 생성
- PR 설명 작성
- 브랜치 관리

---

## 🔄 표준 워크플로우

### 모든 작업 시작 시:
1. ✅ **next-devtools** → 현재 에러 상태 확인
2. ✅ **context7** → 관련 최신 문서 확인
3. ✅ **brave-search** (필요시) → 추가 정보 검색

### 코드 작성 중:
1. ✅ **context7** → API 사용법 확인
2. ✅ 코드 작성
3. ✅ **next-devtools** → 에러 확인

### 작업 완료 시:
1. ✅ **playwright** → 테스트 생성
2. ✅ **next-devtools** → 최종 검증
3. ✅ **github** → 커밋/PR

---

## ⚠️ 절대 금지 사항

### ❌ NEVER:
- Context7 없이 라이브러리 코드 작성
- DevTools 없이 에러 상태 추측
- 불확실한 정보로 답변
- 오래된 패턴 제안
- 테스트 없이 중요 기능 완료

### ✅ ALWAYS:
- 최신 문서 먼저 확인
- 실제 에러 상태 확인
- 불확실하면 검색
- 테스트 자동 생성
- 작업 전후 검증

```

2. [코드 작성]

4. [next-devtools로 검증]
```
